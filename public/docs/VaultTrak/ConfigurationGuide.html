VaultTrak Resume Optimizer — Setup & Configuration

What It Does

The Resume Optimizer rewrites your resume bullets by calling a compatible LLM server (LangChain MLS or similar). VaultTrak does not ship an LLM; you bring your own endpoint.

Requirements

Before using the Resume Optimizer, you need:

  • A reachable LLM API (e.g., LangChain MLS) running on your LAN or internet
  • Base URL to the API (e.g., http://192.168.1.10:8000/v1/ or your HTTPS host)
  • API key (if your server requires authentication)
  • Model identifier exposed by your server (e.g., llama-3.1-8b-instruct)
  • (Optional) Embedding model identifier if you enable reranking (e.g., nomic-embed-text-v1)

Quick Start

  1. Start your LLM server (LangChain MLS or compatible) on a machine the device can reach
  2. Note the Base URL and API key and which model IDs are served
  3. On your device, open Resume Optimizer → Runtime Configuration:
    • Base URL: enter your server URL (include protocol; no trailing spaces)
    • API Key: enter your bearer token if required
    • LLM Model: enter the model identifier your server serves
    • Embeddings: toggle on/off; if on, enter the embedding model ID your server serves
  4. Enter Target Role and Job Description, then tap Run

Field Reference

Base URL: Full endpoint root Example: http://<host>:8000/v1/ API Key: Bearer token if your server enforces auth Leave empty if not required LLM Model Identifier: Name your server exposes for text generation Example: llama-3.1-8b-instruct Embedding Model Identifier: Only needed if reranking is enabled Example: nomic-embed-text-v1

Troubleshooting

  • 401/403 Unauthorized: Wrong/missing API key or your server doesn't expect the auth header
  • Connection errors: Device can't reach the host (Wi‑Fi/VPN/firewall). Verify Base URL and connectivity
  • HTML error page shown: The server returned an error (often 403). Check auth and URL
  • Empty output: Server is up but the model isn't loaded or the model ID is wrong
Important Notes:
• Data flows to your configured LLM endpoint; no offline model is bundled
• Use HTTPS and strong API keys for remote endpoints
• If you switch servers or rotate keys, update Base URL/API Key in the Runtime Configuration

1.2 Sync Settings

Enable cloud sync to keep your data synchronized across devices:

  1. Go to Settings → Sync & Backup
  2. Toggle "Cloud Sync" to ON
  3. Choose sync frequency (Real-time, Hourly, or Manual)
  4. Enable "Auto-backup" for daily backups
Note: Cloud sync requires an active subscription. Free tier users can manually export data.

8. Troubleshooting

Common Issues

  • Sync not working: Check internet connection and sync settings
  • Timer not starting: Grant necessary permissions in device settings
  • Data missing: Ensure latest backup was successful
  • Export fails: Check storage permissions and available space

Reset Settings

If you need to start fresh:

  1. Go to Settings → Advanced
  2. Tap "Reset Settings"
  3. Choose what to reset (settings only or all data)
  4. Confirm action
